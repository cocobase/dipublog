将Hexo博客部署到GitHub Pages

Hexo是功能强大的静态网站生成器，可以和Github Pages结合使用以运行个人博客。这种配合可以帮助我们建立一个方便、高效且免费的博客平台。

背景

Github Pages 内容可以来自于以下三个位置之一：

  • .github.io 仓库分支
  • gh-pages 存储库中的分支
  • 一个docs子目录

现在的大多数教程都会让您根据第一个选项创建新的存储库。如果这样做，您发布的项目就会需要有两个存储库

  • projectname (存储Hexo含有源代码文件夹、主题、配置文件等)
  • websitename (由Hexo生成的静态网站名称，一般位于Hexo生成的博客站点目录下的public目录)

默认情况下，Hexo会将内容生成到名为public的文件夹中。这个文件夹的名称可以通过配置文件_config.yml中的public_dir进行设置。我们可以方便地将目录名改为docs，这样我们就可以匹配在Gitpub Pages中规定的从一个存储库的docs子目录中获取内容。

这种方式还有一个巨大的优点，就是允许您使用Github Pages上的不同存储库发布其它静态网站。

但是，如果您仔细查看Github Pages文档，则会看到有一个限制：

如果您的网站具有名为.github.io 或者 .github.io的存储库，您就无法从其它位置发布您的站点源文件。

如果你在阅读文档时候眨眨眼就错过了这个提示，一旦你用户和组织的githu.io存储库，您就无法在其它任何存储库上使用docs/文件夹来发布内容。因此，您需要确保禁止使用用户和组织上的存储库。
